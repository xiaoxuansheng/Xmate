# XMate 开发流程

> 版本：v2.0 | 更新：2026-07-03

## 开发原则

1. **小步快跑** — 每次只做一个问题，确认可用后再做下一个
2. **先核心后业务** — 核心框架稳定了，再往上面加插件
3. **多问少猜** — 需求不明确时停下来问，不要自行发挥

## AI 助手工作规范

详见 [CLAUDE.md](../CLAUDE.md) 的"工作规范"章节。简要总结：

- **改前**：备份 lib/，聚焦当前问题，只在必要时读 devlog/docs
- **改后**：更新 test list.md（一句话 + 文件 + 3-5 条测试），Devlog 在稳定版本统一整理
- **版本号**：前两位用户决定，第三位自动递增

## 代码审查清单

每次改完自查：

- [ ] 代码符合 [code-standards.md](code-standards.md)
- [ ] 文件位置符合 [project-structure.md](project-structure.md)
- [ ] 公共 API 有文档注释
- [ ] 没有硬编码的路径/密钥
- [ ] 错误有适当处理（至少 log）
- [ ] 新增依赖已记录在 [tech-stack.md](tech-stack.md)
