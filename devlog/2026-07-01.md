## V2.6.1 — 英文系统 translate server 路径 + 版本号不显示修复 🐛

### 问题 1 — 英文系统 translate server 启动失败

英文系统上 Translate server 启动失败："Failed to start: ..." / 找不到 libretranslate。

**根因**：`PythonService.detect()` PATH 命中时 `exePath` 为 `null` → `_cachedPy` 永不设置 → `libretranslatePath` 无法推导 Scripts 目录 → `Process.start` 找不到 `libretranslate.exe`。

**修复**：
- `python_service.dart` `detect()` PATH 命中时：先 `where python` 解析完整路径，再用 `sys.executable` 回退
- 新增 `_queryPythonExecutable()` 辅助方法
- `server_manager.dart` `start()`：`Process.start` 加 `runInShell: true` 安全网

### 问题 2 — 设置页版本号安装后不显示

安装后运行 exe，设置页 General 标签底部显示 "XMate V?.?.?" 而非真实版本号。

**根因**：`_readVersion()` 用 `File('pubspec.yaml').readAsStringSync()` 从 CWD 读取，但 `pubspec.yaml` 不打进安装包。

**修复**：
- `pubspec.yaml` 加入 `flutter.assets` 列表
- `_readVersion()` 改为 `rootBundle.loadString('pubspec.yaml')`（Flutter asset 读取）
- 删除 `dart:io` import
