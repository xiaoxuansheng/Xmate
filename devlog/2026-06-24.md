## V2.1.8: 文本拖拽 → Translate entry 修复

**根因**：Flutter TextField 原生支持拖放文本，当用户从其他应用拖文字到输入框时，Flutter 直接插入文本到 `TextEditingController`，触发 `_search()`，但 `_pasteDetected` 只在 Ctrl+V 时设置，导致拖放路径不会生成 Translate entry。

**修复**：`command_palette.dart` `_search()` 中新增 `textJumped` 检测 — 比较当前查询与上次查询的长度差。正常打字一次只改变 ±1 字符，粘贴/拖放会 >1 字符跳变。同时保留原有 `_pasteDetected` 检测（Ctrl+V 路径）。

```dart
// 保存上轮查询
final prevQ = _lastQ;
_lastQ = q;
// 检测文本跳变（粘贴/拖放 vs 正常打字）
final textJumped = q.isNotEmpty &&
    ((prevQ.isNotEmpty && (q.length - prevQ.length).abs() > 1) ||
     (prevQ.isEmpty && q.length > 1));
// 粘贴或拖放 → 生成 Translate entry
if (q.isNotEmpty && !q.startsWith('=') && (_pasteDetected || textJumped)) {
```

## V2.1.9: 文件拖拽 → 子菜单 (路径直传)

**根因**：`_handleFileDrop()` 把拖入的路径字符串拆成 `rootPath/name/ext/path/isDir` 再构造 `FileSearchResult`，中间环节有 bug：

1. 无扩展名文件（如 `hosts`、`README`）被 `!baseName.contains('.')` 误判为 `isDir=true`
2. 含点号的目录（如 `MyProject 15.0`）被拆成 `name=MyProject 15` / `ext=0`
3. 子菜单只需要 `fullPath` 字符串，构造 `FileSearchResult` 完全多余

**修复**：
- `_enterFileSubmenu(FileSearchResult)` → `_enterFileSubmenu(String filePath)` — 参数直接取绝对路径
- `_handleFileDrop()` 简化为：单文件 → 直接传路径（`/` → `\` 归一化）；多文件 → 静默返回（不做任何事）
- 3 个 `_FileResultEntry` 调用处改为 `_enterFileSubmenu(fr.fullPath)`
- 删除整个路径拆分 + FileSearchResult 构造逻辑

**改动的文件**：`lib/core/command/command_palette.dart`
- `_enterFileSubmenu` 签名（第 437 行）
- `_handleFileDrop` 完全重写（第 793-808 行）
- 4 处调用点更新

## V2.1.10: 文件子菜单 copy/cut/openAsAdmin 失效修复

**根因**：CLAUDE.md 注明 file_operations.cpp 已有 `/` → `\` 路径归一化，但实际代码中未实现。`FileSearchResult.fullPath` 用 `/` 拼接路径，传到 C++ 后直接调 Windows API。`CF_HDROP`（copy/cut）和 `ShellExecuteExW runas`（openAsAdmin）对 `/` 分隔符不兼容，静默失败。

**修复**：
- `flutter_window.cpp` `getPath` lambda：新增 `for (char& c : path) { if (c == '/') c = '\\'; }`归一化（所有 fileops 唯一入口）
- `file_submenu_service.dart` `_execBuiltin`：统一使用 `winPath` 归一化路径（原来仅 `openFolder` 做了）

**改动的文件**：

## V2.1.13: 临时提权统一 — runas 启动新进程

**根因**：`ShellExecuteExW(runas)` 不能直接打开文档文件（返回 SE_ERR_NOASSOC）。Windows 不支持把运行中的普通进程提升，唯一正确方式：`runas` 启动自己的新进程 → UAC → 新进程已提升 → 用默认 verb 打开文件。

**修复**：
- `main.cpp`: 新增 `--open-as-admin` handler（在单实例 guard 之前），提取目标文件路径后用 `ShellExecuteExW` 默认 verb 打开
- `file_operations.cpp` `OpenFileAsAdmin`: 重写为 `runas` 自己 + `--open-as-admin "<path>"` 模式（与 ToggleAutoStart 一致）
- 删除 PowerShell 方案和 debug 日志（`DebugHexDump`、`xmate_oaadebug.txt`）
- `flutter_window.cpp`: 删除 V2.1.11 的 debug hex dump

**涉及文件**：
- `windows/runner/main.cpp` — 新增 handler + `#include <shellapi.h>` + `<string>`
- `windows/runner/native/file_operations.cpp` — 重写 `OpenFileAsAdmin` + 清理
- `windows/runner/flutter_window.cpp` — 删除 debug hex dump

**修复**：
- `file_operations.cpp`: 新增 `RunCommandAsAdmin(cmdPath, args, workDir, hwnd)` — 构建 JSON `{"p":"...","a":"...","d":"..."}` → `ShellExecuteExW(runas, xmate.exe, --run-command "<json>")`
- `file_operations.h`: 声明 `RunCommandAsAdmin`
- `main.cpp`: 新增 `--run-command` handler（JSON 解析 + `ShellExecuteExW` 执行）
- `flutter_window.cpp`: 新增 `runCommandAsAdmin` method channel handler
- `file_submenu_service.dart`: `_execCustom` 中 `runAsAdmin` 改为调用 method channel，与 `runSilently` 分离

**统一后的三个提权入口**：
| 入口 | 命令参数 |

| 开机自启 toggle | `--toggle-autostart` |
| OpenAsAdmin | `--open-as-admin "path"` |
| 自定义命令 runAsAdmin | `--run-command "{\"p\":\"...\",\"a\":\"...\",\"d\":\"...\"}"` |

## V2.3.1: LibreTranslate Server 管理 + 模型管理

### 目标
在 Translate Settings 中实现本地 LibreTranslate 服务器的启动/停止，以及语言模型的安装/卸载管理。

### 架构
```
Dart UI (translate_settings.dart)
  ├─ ServerManager        — 子进程生命周期 (start/stop/health/poll)
  └─ ModelManager         — JSON-line Dart ↔ Python 脚本通信
       └─ translate_model_manager.py  — argostranslate.package API
```

### 新增文件

**`lib/plugins/translate/server_manager.dart`** — 服务器进程管理
- `ServerState` 枚举: unknown/notInstalled/stopped/starting/running/stopping/error
- `Stream<ServerState>` 广播状态变更，UI 实时响应
- `start(loadOnly:)` → `Process.start('python', ['-m','libretranslate',...])` → PID 写入文件
- `stop()` → SIGTERM → `taskkill /F` 兜底 + 删 PID 文件
- `adoptIfRunning()` → 健康检查现有端口，复用孤儿进程
- 10s 间隔健康轮询，自动检测失联

**`lib/plugins/translate/model_manager.dart`** — Dart 封装
- `ModelInfo` / `AvailablePair` 数据类
- `listInstalled()` / `listAvailable()` / `install()` / `uninstall()` / `checkInstalled()`
- `pythonPath` 扫描 `%LOCALAPPDATA%` 下 Python 版本

**`scripts/translate_model_manager.py`** — Python 模型管理脚本
- JSON-line 输出协议：进度 `{"status":"..."}` + 结果 `{"ok":true/false,...}`
- 命令：`list-installed` (含 file size) / `list-available` / `install <from> <to>` / `uninstall <from> <to>` / `index-update` / `check-installed`
