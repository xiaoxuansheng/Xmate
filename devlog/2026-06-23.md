# 2026-06-23 — V2.0.0: File Search 完整功能发布

## 版本概述

V2.0.0 完成 File Search 全部核心功能：过滤器 + 优先级 + 子菜单 + 最近打开。
搜索和截图两个主要插件已全部完成，后续进入新插件开发和优化阶段。

## 新增于 V2.0.0（相对 V1.8.x）

### File Search 过滤器
- 5 个内置关键词过滤器：folder / doc / pic / video / audio
- 输入 `keyword ` 进入过滤模式，仅显示文件结果
- Backspace 在空格边界吞键退出，保留空格
- 自定义过滤器：Keyword / Name / Path / Extensions / Regex
- 设置 UI 支持增删改

### File Search 优先级
- 三档：prefer（+0.50）/ uncommon（×0.3）/ exclude（跳过）
- 12 条默认规则：8 条系统目录 + 4 条文件名正则
- 路径 + 正则 AND 匹配，自定义增删改

### File Search 子菜单
- 9 个内置操作：打开文件夹/复制路径/复制/剪切/创建快捷方式/删除/属性/固定到开始/以管理员打开
- 方向右键/鼠标右键进入，Left/Esc 退出
- 双缓冲模式，与搜索子菜单一致
- 自定义 action：Title / Shortcut / Path(文件选择器) / Args({file}) / Admin / Silent
- 快捷键采集输入 + 全局快捷键执行
- 原生 C++ 文件操作：CF_HDROP / IShellLink / SHFileOperation / ShellExecuteEx / IContextMenu

### 系统图标
- SHGetFileInfoW + SHGFI_SMALLICON + GDI+ → PNG 懒加载

### 默认索引目录
- 首次运行种子：桌面 / 下载 / 文档 / Start Menu / 非C盘

### 打开历史优先
- `markOpened()` 记录到 `_recentOpenSet`，搜索评分 +0.20

### File Search 子选项卡
- Index / Filters / Priority / Submenu 四个子标签

### Commands 增强
- 快捷键采集输入替代手动输入
- 全局系统级热键注册
- 列表显示快捷键 + admin/silent 标签

### 系统
- 管理员自提权启动
- Task Scheduler 开机自启（最高权限）

## 编译

✅ `flutter build windows --debug` 通过

# 2026-06-23 — V2.1.5: 搜索评分优化 + Filter 重构 + 命令面板计算器

## 版本概述

V2.1.5 优化搜索评分和 Filter 交互，修复增量更新目录丢失问题，并新增命令面板内联计算器。

## 搜索评分

### 移除独立配置项
- 删除 `denyExtensions` / `preferExtensions` / `ignoreRegex` / `preferRegex`（由 PriorityRule 替代）
- `SearchConfig` 精简为仅 `indexPaths` + `maxRecentPaths`

### 评分公式调整
- Pinyin/Name 均权：`pm×0.35 + nm×0.30`（原 `pm×0.40 + nm×0.25`）
- Depth 查表封顶：`[0.05, 0.029, 0.024, 0.021, 0.019, 0.018]`，6 层封顶
- 去除 `extBonus` 和 `preferRegex` 评分分量

### Priority 规则叠加
- Prefer 和 Uncommon 不再互斥，可同时生效
- 同时命中 = `×0.3 + 0.50`
- 默认规则新增 Start Menu prefer，解决 Start Menu 快捷方式被 AppData uncommon 误伤

## Filter 模式重构

### 移除全表扫描
- 删除 `_scanWithFilter()` 方法（~60 行），不再扫描全部条目
- 空搜索词显示 placeholder + filter header，不触发搜索

### 进入/退出状态机
- 进入：键入空格且第一词匹配已知 filter（仅首次空格触发）
- 退出：Backspace + text == `keyword ` → 吞键退出，文本保持 `keyword `

## 增量更新修复

- `RENAME_OLD_NAME + isDir` → D| 前缀立即屏蔽旧目录子树
- `RENAME_NEW_NAME + isDir` → 空 incremental segment 加速 compaction
- 跳过空 S|/D| 前缀，避免根目录直接子目录被误杀

## 性能优化

- C++ `ScanDirectory()` / `QueryUsnJournalWithDirs()` → `std::thread` 后台执行

## 设置

- Debug 标签默认隐藏，General 按钮开关
- Priority 标签新增 "Rules" 按钮

## 命令面板计算器

### CalculatorService (`lib/core/command/calculator_service.dart`)
- 递归下降数学表达式求值器，零外部依赖
- 支持：`+` `-` `*` `/` `%` `^` `()` 一元负号/正号、整数和浮点数
- 自实现 pow/exp/ln，无需 `dart:math`
- 无效表达式/不完整表达式/除零返回 `null`

### 命令面板集成
- `_CalcEntry` 密封类：placeholder（仅 `=`）+ 计算结果
- `=` 单独输入显示 "Calculator" + 提示
- `=expr` 显示 `expr = result`，结果橙黄色（`0xFFFFA726`）高亮
- 点击/Enter 打开 `calc.exe`，右击复制结果到剪贴板
- 计算器条目置顶（Search 上方），与其他结果共存

### Enter 键修复
- Enter handler 移到 text-mode early-return 之前，修复文本模式下不执行首条目的问题

## Translate 插件

### 插件架构
- `lib/plugins/translate/` — `TranslatePlugin` + `TranslatePage` + `TranslateService` + `TranslateSettings`
- 注册于 `main.dart`，命令 aliases：translate / translation / 翻译 / fanyi / fy

### 两种触发方式
1. **命令搜索**：输入 translate/翻译 → 命令匹配 → Enter → 打开翻译窗口
2. **粘贴检测**：Ctrl+V 粘贴文本 → `_TranslateEntry` 显示 → Enter → 翻译窗口 + 原文自动填入

### 翻译界面 (`translate_page.dart`)
- 900×600 居中窗口，标题栏可拖拽（`windowManager.startDragging()`）
- 左右双栏：左侧原文输入，右侧翻译结果
- 语言选择器：Auto/EN/ZH/JA/KO/FR/DE/ES/PT/RU/AR/IT/NL，互换按钮
- 底部：文件上传按钮（占位）+ Translate 按钮（disabled，待 API 接入）
- `TranslateService` 占位类，待 LibreTranslate API 对接

### 设置页重构
- Screenshot 选项卡 → Plugins 选项卡 + Screenshot / Translate 子选项卡

# 2026-06-23 — V2.1.6: 命令面板拖拽交互

## 版本概述

V2.1.6 为命令面板添加 Windows 原生拖拽支持（`DragTarget<Object>`），外部拖入文本或文件到面板上触发不同行为。

## DragTarget 集成 (`command_palette.dart`)

### 文本拖拽
- `_handleTextDrop(text)` → 设置 `_pasteDetected = true` + 替换 TextField 内容
- 自动触发 `_search()` → 显示 Translate entry
- 直接替换已有文本（不追加）

### 文件拖拽
- 单文件：`_handleFileDrop(paths)` → 解析路径为 `FileSearchResult` → `_enterFileSubmenu()`
- 进入文件子菜单（复制路径/打开文件夹/等操作），是否在索引内均可
- 多文件：`onWillAcceptWithDetails` 返回 false → 无反应

## 编译

✅ `flutter build windows --debug` 通过
