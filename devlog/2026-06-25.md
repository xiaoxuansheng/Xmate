# 2026-06-25 开发日志

## 版本 V2.2.6 — 滚动截图 dy 精度修复（两轮）

### 第一轮：移除纵向降采样
- **移除 2× 纵向降采样**：`_toGrayDownsampled` 中 `dh = sh ~/ 2` → `dh = sh`，`sy = dy * 2` → `sy = dy`
- **移除 `* 2` 全分辨率映射**（共 6 处）
- dy 不再量化为偶数，精度从 2-px 步进提升到 1-px 步进

### 第二轮：自适应 maxDy + 方向过滤

**问题分析**：
1. 第一下滚动 dy 偏小：`maxDy = roiH/3` 硬上限排除真实位移 → FFT 被迫选 spurious peak → ZNCC ±5 无法纠正
2. 变向第一下方向反+减半：反方向 ZNCC 有更多重叠样本 + 文本列平均信号周期性 → 错误方向拿更高分

**C++ → Dart 数据流变更**：
- `screenshot_capture.cpp`：`MouseLLProc` 中 `WPARAM` 改为 `HIWORD(p->mouseData)` (wheelDelta)
- `flutter_window.cpp`：`WM_XMATE_SCROLL_CAPTURE` handler 把 wheelDelta 打包进 `EncodableMap` → method channel
- `annotate_page.dart`：`_onScrollCapture(wheelDelta)` 读取参数传给 `addFrame(wheelDelta: wheelDelta)`

**算法变更** (`scroll_screenshot_manager.dart`)：
1. **自适应 maxDy**：`maxDy = max(wheelDelta × 1.5, roiH/3)`，上限 `roiH × 3/4`。wheelDelta=0 时回退到原 `roiH/3`
2. **方向过滤**：`wheelDelta != 0 && candidates > 1` 时，保留 `dy.sign == wheelDelta.sign` 的候选，丢弃反向候选。若全部被过滤则保留全部（fallback）

### 涉及文件
`scroll_screenshot_manager.dart`, `annotate_page.dart`, `screenshot_capture.cpp`, `flutter_window.cpp`

**备份**: `lib_bak_v226_20260625_scrolldy2/`

### 第三轮：2D 梯度 ZNCC 替换 1D 列平均相位相关

**根因**：wheel 方向正确但 dy 不稳定 — 1D 列平均信号丢弃了水平纹理结构，不同位移处 ZNCC 分不出优劣。

**算法重写**：
- **删除** `_phaseCorrelate1D`、`_fft1D`、`_znccRefine`
- **新增 `_gradStrip()`**：垂直梯度条（centered difference → `img[y+1][x] − img[y−1][x]`），强化水平边缘
- **新增 `_znccGrad()`**：2D ZNCC 在 `[−maxDy, +maxDy]` 上逐位移计算，利用全部 w×h 梯度像素
- **local peaks → top-3**：每个 strip 找所有局部峰 → 按 ZNCC 排序 → 只保留 top-3 作为 RANSAC 候选
- **方向过滤内联**：`wheelDelta != 0` 时局部峰检测直接跳过反方向 dy（更早、更严格）
- **RANSAC 后更新 debug kept**：strip 的 kept 标志由其峰是否在 inlier 集中决定

## 版本 V2.2.2 — 滚动截图模式 Phase 1 + 2

### Phase 2: 拼接 + 预览缩略图

**拼接算法** (`scroll_screenshot_manager.dart`):
- 像素叠合（SAD 最小化）：连续两帧做像素差绝对值求和，找到最佳重叠偏移
- 垂直方向: 上一帧底部与下一帧顶部比较，搜索范围 5%–90% 帧高，stride 8 采样加速
- `image` 包 `compositeImage` 合成 → `encodePng` 输出
- 缓存机制: `_stitchedPng` + `_stitchedFrameCount`，帧数不变时直接返回缓存

**预览缩略图** (`_buildStitchedPreview`):
- 位置: 选区右侧 8px
- 尺寸: 高度匹配选区 (60–360px)，宽度按比例 (80–260px)
- 内容: 橙色 header（帧数 + 分辨率） + `Image.memory` 缩略图

**Copy/Save/Pin 改用拼接图** (`exportStitchedPng`)

**穿透方案改进**:
- `setWindowHole` (SetWindowRgn): 在选区切透明洞
- `SendInput` 穿过洞投递滚轮
- 解码后 `setState` 渲染新图，下一帧 `clearWindowHole` 关闭洞 → 无闪旧图

### Phase 1: 入口 + 滚动穿透 + 帧捕获

**滚动截图模式入口 + 滚动穿透 + 帧捕获**

**架构概览**：
- Native C++: `CaptureRect` (区域截图) + `SendMouseWheel` (SendInput 滚轮) + `IdentifyWindowUnderRect` (识别目标窗口)
- Dart 服务: `capture_win32.dart` 新增 3 个 method channel 包装方法
- Dart 状态管理: `scroll_screenshot_manager.dart` (新文件) — 帧存储、状态机、isCapturing 防重入
- UI 集成: `annotate_page.dart` — scroll icon、延迟预设、禁用不兼容功能

**C++ 修改** (`screenshot_capture.cpp`)：
- `CaptureRect(rx,ry,rw,rh)` — BitBlt 固定区域截图，返回 PNG + DPR
- `SendMouseWheel(delta)` — SendInput MOUSEEVENTF_WHEEL
- `IdentifyWindowUnderRect(xmateHwnd,rx,ry,rw,rh)` — EnumWindows 找最大交集窗口，返回 JSON {hwnd,className,title}

**滚动穿透流程**：
1. 滚动事件 → 检查鼠标在选区内 → isCapturing guard

**不兼容功能禁用**：
- 滚轮缩放 → 转发到目标窗口
- OCR/翻译 tool → 工具栏拦截 + T 键禁用
- 刷新/搜图 → 图标变灰 + F 键禁用
- 选区把手 → 隐藏 + Shift+箭头 resize 禁用

**延迟预设**: 80/100/120/150ms 可选按钮，在滚动模式下动态显示于选区浮动栏第二行

## 版本 V2.2.1

### 修复 Translate Server 启动无响应 Bug（两轮修复）

**问题**：Settings → Translate → Start 后，Server 长时间显示 "Starting..."，最终报 "Server started but did not respond"，无任何诊断信息。

**第一轮 — 增强诊断** (`server_manager.dart:42,172,203-209,215,233-259,302-326`)：
- 新增 `_outputLines` 缓冲区捕获 stdout/stderr
- `_waitForReady` 改用 `Future.any` 竞速模式：`_healthCheck()` vs `_process!.exitCode`
- 新增 `_buildDiagMessage()` 提取 stderr 附加到错误消息
- 超时 30s → 60s

**第二轮 — 真正根因** (`server_manager.dart:330-343,93-101,187-200`)：
- 手动测试 `python -m libretranslate` 发现报错：
  `No module named libretranslate.__main__; 'libretranslate' is a package and cannot be directly executed`
- libretranslate 包没有 `__main__.py`，pip 安装的是 `libretranslate.exe` CLI 入口（在 Python `Scripts/` 目录）
- 新增 `libretranslatePath` getter：从 `pythonPath` 推导 `Scripts\libretranslate.exe`
- `start()` 改为直接运行 `libretranslate.exe --host ... --port ...`（不再用 `python -m`）
- `isInstalled()` 同样改用 `libretranslatePath`
- 手动验证 `libretranslate.exe` 正常启动并在 12s 内响应

### 新增：App 启动时自动启动 Translate Server

**实现** (`translate_plugin.dart:45-59`)：
- `TranslatePlugin.onInit()` 中新增 `_autoStart()` 调用
- 非阻塞 fire-and-forget：不影响 App 启动速度
- 仅当检测到已安装模型时才启动
- 传入 `loadOnly` 仅加载已安装的语言对
- 静默失败，不打扰用户

### 修复：File Search 符号链接重复结果

**问题**：搜索 `Start Menu/程序/` 下的文件会出现重复结果 — 一条路径含 `程序`，另一条含 `Programs`。

**根因**：Windows `Start Menu/程序` 是 `Start Menu/Programs` 的 NTFS 符号链接 (`lrwxrwxrwx`)。C++ 扫描器递归遍历两个路径，同一个物理文件被索引两次。`normalizePath` 只是做 `/` 归一化和去尾 `/`，无法区分符号链接。

**修复** (`file_search_service.dart:86-100, 742, 849-863, 869-882`)：
1. 新增 `_resolveForDedup(String path)` — 调用 `File.resolveSymbolicLinksSync()` 解析到规范物理路径，带 `_dedupCache` 缓存（同目录下多文件共享父路径）
2. `search()` 入口 `_dedupCache.clear()`
3. 去重 map key 从 `normalizePath(fp)` 改为 `_resolveForDedup(fp)`
4. "fill to 20" 补充逻辑同样使用 `_resolveForDedup` 做 seen set

**备份**：`lib_bak_v221_20260625/` + `lib_bak_v221a_20260625/`

### 修复：首次安装快捷键不生效

**问题**：全新安装后 Alt+Space 和 Alt+S 都不生效，必须先进设置页点一下快捷键捕获后才能用。

**根因分析**：

| 热键 | 原因 |
|------|------|
| Alt+Space | `_paletteKeyId` 硬编码默认值 **62**，但 `LogicalKeyboardKey.space.keyId` = **32**。`findKeyByKeyId(62)` 映射到未知键，注册后实际上不是 Space |
| Alt+S | `main.dart` 只从 `settings.get('screenshot.hotkeyMods')` / `settings.get('screenshot.hotkeyKeyId')` 加载，首次安装两个都是 null → `_ssMods/_ssKeyId` 保持 null → `_registerAllHotkeys` 跳过截图热键注册 |

**修复** (`main.dart:30,183-197`)：
1. `_paletteKeyId` 默认值: `62` → `LogicalKeyboardKey.space.keyId` (32)
2. 截图热键加载：设置不存在时回退到 `ssPlugin.hotkeyMods` / `ssPlugin.hotkeyKeyId`（插件内置默认 Alt+S）
