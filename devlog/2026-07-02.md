## V2.7.3 — 关闭按钮数据丢失修复 + 默认热键修正

### 问题

1. **默认热键错误**：`_kDMods=5, _kDKeyId=0x16`（Alt+Shift+S），正确应为 Alt+R（`1, 0x15`）。`defaultHotKeys` 内 modifiers 也写错为 `[1,2]`（Alt+Ctrl）
2. **关闭按钮导致文件损坏**：录制中/暂停中点 ✕ → `stop()` + 200ms → `windowManager.close()` → `dispose()` → `_ffmpeg?.kill()`。ffmpeg 还未来得及写 trailer 和 concat 就被杀，输出文件残缺/丢失
3. **状态清理验证**：关闭后 `_ipc.dispose()` 删 IPC 文件 → 主进程 `IpcClient` 轮询检测消失 → `_onEngineDisconnected` → `_engine=null, isRecording=false`。下次 Alt+R 正常 spawn，无需改动

### 修复

**screenrecording_plugin.dart**：
- `_kDMods=5, _kDKeyId=0x16` → `_kDMods=1, _kDKeyId=0x15`（Alt+R）
- `defaultHotKeys` modifiers: `[1, 2]` → `[1]`

**screenrecording_app.dart** `onClose` 回调：
- 录制中/暂停时 `_service.stop()` → 轮询 `_status`（200ms × 60 = 12s）等待 `stopped` 或 `error` → 再 `windowManager.close()`
- 空闲/已完成状态直接关闭（无延迟）

**状态清理链路**（验证通过，无代码改动）：
```
子进程 close → stop+finalize → dispose → _ipc.dispose() 删 IPC 文件
  → 主进程 IpcClient 检测消失 → _onEngineDisconnected → _engine=null, isRecording=false
  → 下次 Alt+R → hasEngine=false → 全新 spawn
```

## V2.7.1 — Screen Recording 指示条重构

### 概述

录制指示条 UI 完全重新设计，引入暂停/继续支持（FFmpeg 分段+concat），新增内联设置面板和录制完成文件条。

### 改动

**数据模型：**
- `RecordingStatus` 新增 `paused` 状态
- 新增 `RecordingSegment` 类（`{path, startMs, endMs}`）跟踪分段

**录制服务 (recording_service.dart)：**
- `pause()`: 发 `q` 到 ffmpeg stdin → 停止当前段 → 记录 endMs
- `resume()`: 启动新 ffmpeg 进程写 `_part{N}.mp4` → 新 segment
- `stop()`: paused 状态直接 `_finalizeSegments()`；recording 状态发 `q` → 等 ffmpeg exit → concat
- `_finalizeSegments()`: 单段 rename → 输出路径；多段 `ffmpeg -f concat` 合并 → 删临时文件
- `fileSize`: 录制完成后读取输出文件大小

**指示条 UI (recording_indicator.dart)：**
- 固定宽度的状态区(72px) + 计时区(52px)，按钮不会再因数字宽度变化移位
- 状态：● IDLE(灰) / ● REC(红色闪烁 600ms) / = PAUSE(蓝) / ✓ SAVED(绿) / ⚠ ERROR(红)
- 5 按钮：录制(● idle/■ recording) / 暂停(⏸/▶) / 键鼠(预留) / 设置(⚙) / 关闭(✕)
- 文件条：文件名 + fileSizeStr + 打开文件夹/删除/关闭 三按钮
- 背景始终 `0xDD1A1A2E`（不再按状态变色）

**内联设置面板 (recording_settings_panel.dart)：**
- Encoder (Dropdown) + FPS (Dropdown) + CRF (Slider)
- 变更直接写 `SettingsService`（key 前缀 `screenrecording.`）
- 通过 `ScreenRecordingApp` 的 `_showSettings` 切换显示
- 窗口随面板高度动态 resize

**子进程 App (screenrecording_app.dart)：**
- 新的按钮回调：`_onRecord`（idle→start / rec→stop）、`_onPause`（rec↔paused）、`_onToggleSettings`
- `_currentSettings`（encoder/fps/crf）初始从 SrData，可被 settings panel 修改
- `_onOpenFolder`: `explorer /select,` 定位文件
- `_onDeleteFile`: 删除输出文件
- `_updateWindowSize()`: Settings 面板+120px，File bar+42px

### 已知问题

1. ffmpeg.exe (v8.1.2, ~97MB) 已捆绑，POST_BUILD 复制正常
2. 编译+启动通过，xmate.exe 正常启动无崩溃

### 根因

1. **路径未归一化**：Dart 传 `C:/Users/...` (forward slash)，但 `ScanDirectoryAsync` 直接传给 `ScanDirectory` → `FindFirstFileW`。Windows 10+ 大部分情况容错 `/`，但非 100% → 扫描静默返回 `"[]"` → Dart 收到空数组 → "No files found"。

2. **`g_engineAlive` 反模式**：第一轮加的 `atomic<bool>` 理论上初始化 `true`，但 detached 线程 + atomic flag 是脆弱的 anti-pattern。线程完成扫描后检查 flag → 若为 false 则跳过 messenger invoke → Dart completer 永远不 resolve → 重建在 "Scanning..." 处永久挂起。

3. **Dart 无 timeout**：`completer.future` 无超时保护 → 任何 C++ 线程问题导致回调丢失 → 永久挂起。

### 修复

| 文件 | 修改 |
|------|------|
| `file_scanner.cpp` | `ScanDirectoryAsync` 入口归一化 `/` → `\`；删除 `g_engineAlive` 和 `#include <atomic>` |
| `file_scanner.h` | 删除 `SetFileSearchShuttingDown()` 声明 |
| `usn_journal.cpp` | 删除 `extern g_engineAlive` 和 `#include <atomic>` |
| `flutter_window.cpp` | `OnDestroy` 删除 `SetFileSearchShuttingDown()` |
| `file_search_channel.dart` | `scanDirectoryAsync` 的 completer 加 5 分钟 `.timeout()` |

## V2.6.3 第一轮 — 退出清理修复（DLL 占用解除）

### 概述

修复退出后残留线程/COM 对象/全局钩子/Pin 窗口/GDI+ 等导致卸载时大量 DLL 无法删除的问题。共 C++ 10 处修复 + Dart 1 处修复。

### 问题清单与修复

| 优先级 | 问题 | 修复 |
|--------|------|------|
| **P0** | WH_MOUSE_LL 全局钩子未卸载 → host EXE/DLL 锁在所有 32 位进程 | `OnDestroy` 中调 `UninstallScrollHook()` |
| **P0** | `file_scanner.cpp:290` / `usn_journal.cpp:523` `std::thread::detach()` → 线程在 DLL 卸载后仍可能执行 | 新增 `std::atomic<bool> g_engineAlive`，`OnDestroy` 设置 `SetFileSearchShuttingDown()`，线程检查标志后跳过 messenger invoke |
| **P1** | Pin 窗口未销毁 + GDI+ 未 shutdown → gdiplus.dll 无法卸载 | 新增 `DestroyAllPinWindows()`：销毁所有 Pin HWND + `GdiplusShutdown` |
| **P1** | Office 预览 COM handler + proxy pool 未释放 → ole32.dll 持有引用 | 新增 `DestroyAllWordPreviews()`：Unload + Release handler + 清空 proxy pool |
| **P1** | dragout handler 内 `OleInitialize` 不配对 | `S_OK==OleInit` → 返回前 `OleUninitialize` |
| **P1** | Dart `FileSearchService._autoUpdateTimer`（1 分钟）+ `ServerManager._healthTimer`（10 秒）未取消 | main.dart exit 中加 `FileSearchService().dispose()` + `ServerManager().dispose()` |
| **P2** | Main mutex handle 未关闭 | `CloseHandle(hMutex)` 在消息循环结束后 |
| **P2** | 文本拖出时 pointer move 覆盖选区 | `freezeDragState()` 在 `onPointerDown` 冻结 `_dragTextSnapshot` |
