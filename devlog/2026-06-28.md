## 版本 V2.4.2 — QuickLook 独立进程 + 底层修复 ⭐

### 核心架构：独立进程模式

QuickLook 从单进程 Overlay → 独立进程（`xmate.exe --quicklook <path>`）。

```
主进程 Alt+Q
  → COM getExplorerSelection
  → closeQuickLookWindows (关闭旧预览)
  → Process.start(xmate.exe --quicklook path, detached)
       ↓
  main.cpp 检测 --quicklook
    → 跳过 Mutex/托盘/服务
    → FlutterWindow("quicklook", path)
    → WM_ACTIVATE guard (拦截程序化激活)
    → QuickLookApp → QuickLookPage
    → showNoActivate (SWP_NOACTIVATE)
    → 400ms 轮询 Explorer 选中变化
```

### 底层关键修复

#### 1. window_manager 可见性状态机
`waitUntilReadyToShow` 回调调了 `hide()` → window_manager 内部 `_isWindowVisible = false` → `startDragging`/`setAlwaysOnTop`/`setSize` **全部短路**。
修复：回调改为空操作，`_onReady` 先 `showNoActivate` 再 `windowManager.show()` 同步标志。

#### 2. 焦点不被抢 (WM_ACTIVATE guard)
独立窗口必定走 `ShowWindow(SW_SHOW)` → `WM_ACTIVATE`。C++ 层拦截：
- `WM_ACTIVATE` 非 `WA_CLICKACTIVE` → `return 0` 吃掉
- `WM_MOUSEACTIVATE` → `MA_ACTIVATE`（用户点窗口才激活）
- Dart 不调 `setAlwaysOnTop`（GetWindowPos HWND_TOPMOST 也触发激活）

#### 3. CloseQuickLookWindows 单实例
`EnumWindows` + `GetWindowTextW("xmate_ql")` + `SendMessage(WM_CLOSE)` — 同步关闭，再 spawn 新进程。

#### 4. 退出主进程清理子进程
`onExit` 回调开头加 `closeQuickLookWindows` channel 调用。

### 功能优化

| 功能 | 实现 |
|------|------|
| Hotkey 设置 | QuickLookSettings（快捷键采集器，冲突检测，持久化到 `quicklook.hotkeyMods/KeyId`） |

| Q 键 keyId | `LogicalKeyboardKey.keyQ.keyId`（非硬编码 0x14） |
| Pin/Fullscreen/OpenWith | 原生 C++ `SHOpenWithDialog` |

## V2.4.3: QuickLook Alt+Q Toggle 修复

### 问题
Alt+Q 关闭 QuickLook 窗口后，立刻又打开一个新窗口。根因：`_showQuickLook()` 先调 `closeQuickLookWindows` 关闭旧窗口，然后无条件 spawn 新进程。

### 修复
`lib/main.dart` `_showQuickLook()`:
- 捕获 `closeQuickLookWindows` 返回值（关闭了几个窗口）
- `closed > 0` → `return`（是 toggle 关闭，不开新的）
- `closed == 0` → 获取 Explorer 选中 → spawn 新进程（是 toggle 打开）

## V2.4.3 ~ V2.4.5: QuickLook 焦点排查（未完全解决）

逐层拦截了以下焦点窃取路径，均未能阻止 `fl=0003 → WM_ACTIVATE`：
1. `windowManager.show()` → `SetForegroundWindow()` — 移除调用
2. `Win32Window::SetChildContent` → `SetFocus(child)` — 移除
3. `waitUntilReadyToShow` → `setSize(1,1)` → `SetWindowPos` 缺 `SWP_NOACTIVATE` — 去掉 `size:` 参数
4. `showNoActivate` → `SWP_SHOWWINDOW` + `HWND_TOPMOST` — 改用 `HWND_TOP` / `SWP_NOZORDER`
5. `WM_WINDOWPOSCHANGING` 强补 `SWP_NOACTIVATE` — 进入时 fg 已切换，太晚
6. `ImmAssociateContext(nullptr)` + 拦截 `flutter/textinput` channel — 无效
7. `WM_IME_SETCONTEXT` 拦截 — 无效

根因：`flutter_windows.dll` 内部在引擎初始化期调用 `SetWindowPos` 不带 `SWP_NOACTIVATE`，栈帧指向 `flutter_windows.dll` 内部，无法修改编译后的 DLL 源码。

## V2.4.6: 焦点回还 + 代码清理 ⭐

### 策略转变
不再拦截焦点被抢，改为抢完后归还。主进程 spawn 子进程时捕获当前前台 HWND（Explorer），子进程 `_onReady` 完成后 `AllowSetForegroundWindow + SetForegroundWindow` 还给 Explorer。

### 数据流
```
主进程: getForegroundHwnd → hex → --ql-restore <hex>
  → 子进程 main.cpp: 解析 --ql-restore → FlutterWindow 构造函数
  → app_channel "getQlRestoreHwnd" → Dart read → QuickLookApp.restoreHwnd
  → _onReady → showNoActivate (resize) → window channel "restoreForeground"
  → AllowSetForegroundWindow(ASFW_ANY) + SetForegroundWindow(Explorer)
```

### 代码清理
- 删除所有诊断日志（`_QLogMsg` namespace / stack traces / `WM_WINDOWPOSCHANGING` hacks / `WM_IME_SETCONTEXT` intercept）
- 删除 `ql_textinput_channel_` / `#include <imm.h>` / `ImmAssociateContext` 等无效尝试
- 恢复 `SetChildContent::SetFocus(child)` — 对主进程必要
- 恢复 NextFrameCallback `SetWindowPos(child, HWND_TOPMOST, ...)` — 回到 V2.4.2 基线

### 修改文件
| 文件 | 改动 |
|------|------|
| `flutter_window.h` | 新增 `ql_restore_hwnd_` + 构造函数 param |
| `flutter_window.cpp` | 清除诊断 + `app_channel_` 新增 `getForegroundHwnd`/`getQlRestoreHwnd` + `window_channel_` 新增 `restoreForeground` |
| `main.cpp` | 解析 `--ql-restore <hex>` → `FlutterWindow` constructor |

## V2.4.7: 命令面板 → QuickLook 文件预览联动 ⭐

### 核心设计

命令面板打开时自动 spawn QuickLook 子进程；上下箭头导航到文件搜索结果时，QuickLook 窗口实时展示该文件预览。通信通过共享 JSON 文件（`%APPDATA%/XMate/ql_palette.json`）完成。

```
命令面板 ↑↓ 选中文件条目
  → _notifyPaletteState() → QuickLookPaletteState.update(path)
  → %APPDATA%/XMate/ql_palette.json {path, active:true}

QuickLook 进程 (400ms poll)
  → QuickLookPaletteState.read()
  → active && path → 加载预览
  → !active → Explorer COM fallback
```

## V2.4.10 — 三个修复

### 1. QuickLook 命令从面板隐藏

`quicklook_plugin.dart`: `commands => []` — QL 只由 Alt+Q 热键触发，不在命令面板搜索中出现。

### 2. QL 默认初始位置

修复：ImageView / TextView / FallbackView 各加 `didUpdateWidget`，检测 `filePath` 变化 → 清空状态 → `_load()` 重载。
- `ImageView` 额外重置 `_centered = false` 让居中平移重新计算。
